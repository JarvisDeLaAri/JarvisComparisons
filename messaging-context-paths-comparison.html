OpenClaw Communications Paths

Goal: get instructions into the main session, decide whether WhatsApp should speak back, and keep the transcript honest.

🧭 Quick takeaways

  • message tool: writes into the active session transcript as an assistant turn, and can deliver to WhatsApp.
  • sessions_send: injects into the target session, triggers an agent turn, and stays silent unless the target path sends outward.
  • /v1/chat/completions: works only when the gateway endpoint is enabled and auth matches, then it can be pinned to a session key.
  • best combo: cron timing, sessions_send for instruction, message for human-visible delivery.

  • message tool: logged in context, but it is outbound assistant delivery, not a user-role instruction.

🧪 Examples

1) message tool (requested example)

openclaw message send --channel whatsapp --target +972542634114 --message "send me a cow emoji and explain why"

2) sessions_send

openclaw sessions send --session agent:main:whatsapp:direct:+972542634114 --message "send me a sheep emoji and explain why"

curl -sS "$OPENCLAW_URL/v1/chat/completions" -H "Authorization: Bearer $OPENCLAW_TOKEN" -H "Content-Type: application/json" -d '{"model":"openai-codex/gpt-5.3-codex","user":"agent:main:whatsapp:direct:+972542634114","messages":[{"role":"user","content":"send me a sheep emoji and explain why"}]}'
