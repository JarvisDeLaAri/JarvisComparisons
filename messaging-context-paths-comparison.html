OpenClaw Messaging Paths — 4-Way Compare

Message Tool vs Internal sendMessage vs /v1/chat/completions vs sessions_send, with explicit context-awareness rating.

Public Agent Tool

1) message tool

Primary runtime tool for channel actions like send/react/poll, routed through OpenClaw core + plugins.

  • Context awareYES
  • Session continuityNative
  • Direct user deliveryYES
  • Best forNormal agent messaging
  • RiskLow (policy governed)
Internal Runtime Function

2) sendMessage (internal)

Core outbound plumbing used under the hood by tool/plugin layers. Not a normal user-facing orchestration entrypoint.

  • Context awareNO
  • Session continuityNot by itself
  • Direct user deliveryYES
  • Best forCore/plugin implementation
  • RiskHigh misuse potential
HTTP Integration Surface

3) /v1/chat/completions

OpenAI-compatible gateway endpoint. Executes as normal agent run, good for external apps and API integrations.

  • Context awarePARTIAL
  • Session continuityNeeds stable user/session key
  • Direct user deliveryBy flow
  • Best forOpenWebUI / custom clients
  • RiskOperator surface exposure
Cross-Session Handoff

4) sessions_send

Injects a message into a target session and lets that target agent answer with its own context and policies.

  • Context awareYES (target session)
  • Session continuityStrong in target session
  • Direct user deliveryVia target logic
  • Best forIsolated cron -> main handoff
  • RiskWrong target = wrong context

Context-awareness truth table

  • message tool: yes, because it runs in the active agent session transcript.
  • sendMessage internal: no, transport-level function, no conversational brain.
  • /v1/chat/completions: can be yes, but only if session is pinned (stable user/session key). Stateless calls are not context aware.
  • sessions_send: yes, in the target session only. Context follows the destination transcript.

Verdict (Ariel-style)

For your fitness flow: isolated cron dispatcher + sessions_send handoff to main + main uses message tool. That gives silent cron when no shout is needed, and real context-awareness when you reply.
