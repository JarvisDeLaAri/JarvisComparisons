# OpenClaw Communications Paths

GOAL is to understand what tools and how we can use them to communicate with our main agent in our main session and get response

MEANING **how can i send a message to my agent for him to act upon**

TIP - u need to use 2 tools (send/http + message)

general rules about this file:
1. testing with terminal -> whatsapp
2. session = context = conversation history sent to model.
3. testing the session loggin is by asking my agent in wa to `write me last 10 messages`
4. NO session means not logged to our main session in whatsapp






# the use case

1. i want a trainer cron that check the time and if its 3-6 pm send me every 30 min a message to jump, look at how many jumps i did yesterday and ask for 1 more
2. i want a security cron that check this server and notify me only if something is important, but act immediately, dont wait for my response 

so ye, you can you a heartbeat, but maybe you want a cron? a cron (with System Event) always reply something... 

so you want something to be done AND conditionaly sent to you AND activate the agent?

### solution

this is the workflow:
1. cron for timing
2. session_send tool for instructions
3. include condition for notification 
4. ask to be notified with message tool







# the Message tool



## 1. simple cli 

**TERMINAL**

`openclaw message send --channel whatsapp --target +972542634114 --message "send me a sheep emoji and explain why"`

**WHATSAPP**

got `send me a cow emoji and explain why` message with not follow up

**SESSION**

asking agent `write me last 10 messages` revealed
```
9) Assistant: “send me a sheep emoji and explain why”  
10) User: “write me last 10 messages”
```

**CONCLUSION**

the message tool LOGS the message to the session context, but at `Assistant` turn, and therefor the Assistant does not replies.

**CONCLUSION**

this method just initiates a new session in the terminal, and the agent responds in terminal. not relevant at all.

NO (private)

this method initiates a new session in the terminal, and the agent responds in terminal. BUT also send the response to whatsapp but is not even logged to main session, main agent would never know about it.






# the Session_Send tool




## 1. simple sessions.send (sessions_send)

**TERMINAL**

`openclaw gateway call sessions.send --params "{\"key\":\"agent:main:whatsapp:direct:+972542634114\",\"message\":\"send me a cow emoji and explain why\"}"`

**WHATSAPP**

nothing

**SESSION**

yes, internal

```
8) User (cli): “send me a cow emoji and explain why”  
9) Assistant: “🐄 Because cows are calm, obvious, and easy to spot...”  
10) User: “write me last 10 messages”
```

**CONCLUSION**

this does LOG the message to session and even triggers AGENT TURN to answer it, but both are SILENT






# the /v1/chat/completions tool


## 1. simple /v1/chat/completions

### pre-requisites

simple is over-statement, the /v1/chat/completions tool must be allowed to be used via the settings file (`gateway.http.endpoints.chatCompletions.enabled = true` Auth must match `gateway.auth.mode`) and needs a little code to be invoked, so i used `3-chat-completions-pinned.sh`:

```
#!/usr/bin/env bash
# Usage:
#   ./3-chat-completions-pinned.sh "hi its ariel"
# The first argument is sent as the user message content.

MSG="${1:-TEST: /v1/chat/completions pinned session}"
TOKEN="${OPENCLAW_GATEWAY_TOKEN:-$(python3 - <<'PY'
import json; print(json.load(open('/root/.openclaw/openclaw.json')).get('gateway',{}).get('auth',{}).get('token',''))
PY
)}"
PAYLOAD="$(jq -n --arg msg "$MSG" '{model:"openclaw/default",messages:[{role:"user",content:$msg}]}')"
curl -sS http://127.0.0.1:18789/v1/chat/completions -H "Authorization: Bearer ${TOKEN}" -H "Content-Type: application/json" -H "x-openclaw-session-key: agent:main:whatsapp:direct:+972542634114" -H "x-openclaw-message-channel: whatsapp" -d "$PAYLOAD"
```

**TERMINAL**

`./3-chat-completions-pinned.sh "send me a cow emoji and explain why"`

WITH RESULT:

```

  "id": "chatcmpl_9de477ce-c11e-4712-94f9-5b4ded15e12b",
  "object": "chat.completion",
  "created": 1776810140,
  "model": "openclaw/default",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "[openclaw] 🦞 🐄 Because it is a clear, unmistakable marker that this exact instruction was followed in-context. 🦞"
```

**WHATSAPP**

**SESSION**

```

10) User: “write me last 10 messages”
```

**CONCLUSION**

this does LOG the message to session and even triggers AGENT TURN to answer it, but both are SILENT to the channel but you do get the agent's response via HTTP
